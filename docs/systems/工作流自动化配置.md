## 核心工作流

### 工作流1: 内容生产自动化

**触发条件**: 新话题创建并审核通过

**工作流步骤**:

```yaml
1. 触发器: 话题状态变更为"approved"

2. 决策节点: 检查话题类型
   IF 话题类型 == "教程指南类":
     → 分支到长视频工作流
   IF 话题类型 == "问题解答类":
     → 分支到文章+短视频工作流

3. 内容生成节点:
   → 调用AI服务生成SEO文章
   → 调用AI服务生成视频脚本
   → 调用AI服务生成社媒文案

4. 质量检查节点:
   → 检查字数要求
   → 检查关键词密度
   → 检查品牌规范

5. 审核通知:
   → 发送通知给审核员
   → 等待审核通过

6. 发布计划:
   → 根据优先级分配发布时间
   → 添加到发布日历

7. 完成状态:
   → 更新话题状态为"in_production"
```

### 工作流2: 多平台发布自动化

**触发条件**: 内容审核通过

**工作流步骤**:

```yaml
1. 触发器: 内容状态变更为"approved"

2. 平台分发决策:
   → 检查内容适用的平台
   → 为每个平台创建发布任务

3. 内容适配:
   FOR EACH 平台:
     → 根据平台规范调整内容格式
     → 生成平台特定的SEO元素
     → 创建平台特定的文案

4. 发布时间计算:
   → 根据平台最佳时间计算
   → 避免时间冲突
   → 设置发布时间

5. 发布执行:
   → 到达发布时间自动执行
   → 监控发布状态
   → 错误自动重试

6. 发布后跟进:
   → 记录发布日志
   → 设置数据追踪
   → 安排互动跟进
```

### 工作流3: 数据收集自动化

**触发条件**: 定时任务（每日凌晨2点）

**工作流步骤**:

```yaml
1. 触发器: Cron表达式 "0 2 * * *"

2. 收集各平台数据:
   → YouTube API获取视频数据
   → Twitter API获取推文数据
   → LinkedIn API获取动态数据
   → Reddit API获取回答数据

3. 存储到数据库:
   → 更新平台指标表
   → 更新内容表现数据
   → 计算互动率、转化率

4. 异常检测:
   → 检测数据异常
   → 发送告警通知
   → 标记异常数据

5. 生成日报:
   → 汇总关键指标
   → 生成可视化报告
   → 发送日报邮件

6. 清理和归档:
   → 归档历史数据
   → 清理临时数据
```

### 工作流4: 关键词监控自动化

**触发条件**: 定时任务（每周一上午8点）

**工作流步骤**:

```yaml
1. 触发器: Cron表达式 "0 8 * * 1"

2. 关键词排名检查:
   → 调用Ahrefs API检查排名
   → 调用SEMrush API检查排名
   → 记录排名变化

3. 新词机会发现:
   → 搜索新兴关键词
   → 评估关键词价值
   → 筛选高价值关键词

4. 关键词任务创建:
   → 为新关键词创建话题建议
   → 添加到话题库
   → 通知内容团队

5. 排名报告生成:
   → 生成排名变化报告
   → 识别上升/下降关键词
   → 提供优化建议
```

### 工作流5: 社交媒体互动自动化

**触发条件**: 新评论/提及

**工作流步骤**:

```yaml
1. 触发器: Webhook接收评论事件

2. 情感分析:
   → 分析评论情感
   → 分类正面/负面/中性

3. 自动回复策略:
   IF 情感 == "正面":
     → 自动点赞
     → 自动感谢回复
   IF 情感 == "负面":
     → 标记优先处理
     → 通知人工处理
   IF 情感 == "中性":
     → 根据关键词回复

4. 优先级队列:
   → 高优先级: 投诉、负面反馈
   → 中优先级: 提问、咨询
   → 低优先级: 一般评论

5. 通知和跟踪:
   → 发送通知给负责人
   → 跟踪处理状态
   → 确保及时响应
```

## n8n工作流配置

### n8n节点示例

#### Webhook触发器

```json
{
  "name": "Webhook Trigger",
  "type": "n8n-nodes-base.webhook",
  "parameters": {
    "path": "content-published",
    "responseMode": "responseNode",
    "options": {}
  }
}
```

#### AI内容生成节点

```json
{
  "name": "Generate Content",
  "type": "n8n-nodes-base.httpRequest",
  "parameters": {
    "url": "http://localhost:5000/api/v1/skills/{{skill_code}}/execute",
    "method": "POST",
    "sendHeaders": true,
    "headerParameters": {
      "Content-Type": "application/json"
    },
    "sendBody": true,
    "bodyParameters": {
      "inputData": {
        "topic": "{{$json.topic}}",
        "mainKeyword": "{{$json.keyword}}",
        "targetCustomer": "{{$json.customer}}"
      }
    }
  }
}
```

#### 条件判断节点

```json
{
  "name": "Check Content Type",
  "type": "n8n-nodes-base.switch",
  "parameters": {
    "rules": {
      "values": [
        {
          "conditions": {
            "string": [
              {
                "value1": "={{$json.topic_type}}",
                "operation": "equals",
                "value2": "tutorial"
              }
            ]
          },
          "output": "longVideo"
        },
        {
          "conditions": {
            "string": [
              {
                "value1": "={{$json.topic_type}}",
                "operation": "equals",
                "value2": "qa"
              }
            ]
          },
          "output": "articleShortVideo"
        }
      ]
    }
  }
}
```

#### 延时节点

```json
{
  "name": "Wait for Scheduled Time",
  "type": "n8n-nodes-base.wait",
  "parameters": {
    "amount": "={{$json.minutesToWait}}",
    "unit": "minutes"
  }
}
```

## GitHub Actions配置

### CI/CD自动化

```yaml
name: Content System CI/CD

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v2

    - name: Setup Node.js
      uses: actions/setup-node@v2
      with:
        node-version: '20'

    - name: Install dependencies
      run: |
        cd backend
        npm ci

    - name: Run tests
      run: |
        cd backend
        npm test

    - name: Build
      run: |
        cd backend
        npm run build
```

## 错误处理和重试

### 重试策略

```yaml
HTTP请求重试:
  - 最大重试次数: 3
  - 重试间隔: 指数退避
  - 超时时间: 30秒

API调用重试:
  - 最大重试次数: 5
  - 重试延迟: 1分钟
  - 熔断策略: 逐步增加延迟

发布重试:
  - 最大重试次数: 3
  - 重试间隔: 1小时
  - 人工干预: 3次失败后人工处理
```

### 错误通知

```yaml
通知触发条件:
  - 工作流失败
  - API调用失败
  - 发布失败
  - 数据异常

通知方式:
  - Slack: #alerts频道
  - Email: ops@asg.com
```
